## NoSQL概述

非关系型数据库，数据直接无关系，易扩展

* 易扩展
* 大数据高性能
* 灵活的数据模型
* 高可用

与关系数据库相比，NoSql数据库使用不同的数据结构。它使NoSQL中的一些操作更快。

|  |  |
| --- | --- |
| 关系数据库 | NoSQL数据库 |
| 支持强大的查询语言。 | 支持非常简单的查询语言。 |
| 它具有固定的模式。 | 无固定模式。 |
| 遵循ACID（原子性，一致性，隔离性和持久性）。 | 只有“最终一致”。 |
| 支持事务。 | 不支持事务。 |

## NoSQL分类

### 键值存储数据库

主要使用哈希表。优势在于简单易部署。代表为Redis

### 列存储数据库

应对海量分布存储数据。键存在但指向多个列。列由家族来安排，如Cassandra、HBase

### 文档型数据库

与键值存储类似，该类型的数据模型是版本化的文档，半结构化的文档以特定格式存储如JSON。文档数据库可以看作键值数据库升级版，允许嵌套键值，比传统键值数据库查询效率更高。如：CouchDB，MongoDB
