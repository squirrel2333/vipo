
## 数据模型

### 集群（Cluster）

Cassandra数据库是为跨越多条主机共同工作，对用户呈现为一个整体的分布式系统设计的。Cassandra最外层容器被称为集群。Cassandra将集群中的节点组织为一个环，然后把数据分配到集群中的节点上。对于故障处理，每个节点包含一个副本，如果发生故障，副本将复制。

### 键空间（KeySpace）

Cassandra的键空间（KeySpace）相当于数据库，我们创建了一个键空间就是创建了一个数据库。一般将有关联的数据放到同一个KeySpace下。键空间是 Cassandra 中数据的最外层容器。Cassandra 中的一个键空间的基本属性是：

#### 副本因子

决定有几分副本，每个副本位于不同节点之上。所有节点同样重要，没有主从之分

#### 副本放置策略

* 简单策略
* 网络拓扑策略

#### 列族

键空间是一个或多个列族的列表的容器。列族又是一个行集合的容器。每行包含有序列。列族表示数据的结构。每个键空间至少有一个，通常是许多列族。

```
CREATE KEYSPACE Keyspace name
WITH replication = {'class': 'SimpleStrategy', 'replication_factor' : 3};
```

### 列族（column family）

相当于关系型数据库的表（table），是包含了多行（row）的容器

![](Cassandra_数据模型_002.png)

|  |  |
| --- | --- |
| 关系表 | Cassandra 列族 |
| 关系模型中的模式是固定的。一旦为表定义了某些列，在插入数据时，在每一行中，所有列必须至少填充一个空值。 | 在 Cassandra 中，虽然定义了列族，但列不是。您可以随时向任何列族自由添加任何列。 |
| 关系表只定义列，用户用值填充表。 | 在 Cassandra 中，表包含列，或者可以定义为超级列族。 |

Cassandra 列族具有以下属性 -

* **keys\_cached** - 它表示每个 SSTable 保持缓存的位置数。
* **rows\_cached** - 它表示其整个内容将在内存中缓存的行数。
* **preload\_row\_cache** -它指定是否要预先填充行缓存。

**注 -** 与不是固定列族的模式的关系表不同，Cassandra 不强制单个行拥有所有列。

![](Cassandra_数据模型_003.png)

#### ColumnFamily两种类型

* 静态column family：字段名固定，适合column都有预定义的元数据
* 动态column family：字段名是计算出来的，列族只能提供类型不能指定字段名字和值

#### RowKey（行键）

ColumnFamily中每一行都用Row Key标识，相当于关系型数据库中的主键，并且总是被索引的

#### 主键

Cassandra可以使用ＰＲＩＭＡＲＹ　ＫＥＹ关键字创建主键，分为两种：

* Single column primary Key 主键由一列组成
* Composite Primary Key 主键由多列组成

### 列（column）

列是 Cassandra 的基本数据结构，具有三个值，即键或列名称，值和时间戳。下面给出了列的结构。

|  |  |  |
| --- | --- | --- |
| 名称 | 值 | 时间戳 |
| name: byte[] | value: byte[] | clock: byte[] |

Cassandra不需要预先定义列，只需要在KeySpace里定义列族

### 超级列

超级列是一个特殊列，因此，它也是一个键值对。但是超级列存储了子列的地图。

通常列族被存储在磁盘上的单个文件中。因此，为了优化性能，重要的是保持您可能在同一列族中一起查询的列，并且超级列在此可以有所帮助。下面是超级列的结构。

### 集合类型

* set 元素无序
* list 有序列表数据，默认按照插入顺序保存
* map包含 key/value键值对

注意：

* 使用集合，每一项最大64K
* 保持集合内数据不要太大，免得时间过长
* 不要向集合插入大于64k，否则后面的会丢失
