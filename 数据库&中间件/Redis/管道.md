# 与原生批量命令对比

* 原生批量命令是原子性（mset，mget），pipeline是非原子性
* 原生批量命令一次只执行一种命令，pipeline支持批量执行不同命令
* 原生批命令是服务端实现，pipline需要服务端客户端共同完成

# 与事务对比

* 事务具有原子性，pipeline没有
* 管道一次发送多条到服务器，事务一条一条发
* 执行事务会阻塞其它命令，执行管道不会

# 事项

* 依此执行，不保证原子性，过程中指令异常，会继续执行后续
* 命令不能太多，不然客户端等待时间过久，服务端被迫回复队列答复，占用很多内存
