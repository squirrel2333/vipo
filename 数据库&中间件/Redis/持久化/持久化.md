# 同时使用RDB/AOF

RDB是默认，如果AOF被开启，重启时会优先加载AOF。既存在aof时不会加载RDB

![](持久化_002.png)

同时开启两者：

* redis会优先恢复AOF文件数据，因为AOF数据要比RDB数据完整
* 不建议只使用AOF，因为RDB更适合备份数据库（aof在不断变化不好备份）
* 所以推荐混合使用，RDB镜像做全量持久化，AOF做增量持久化：先使用RDB进行快照存储，AOF记录写操作，当重写策略满足或手动触发重写时，将最新的存储数据存储为最新的RDB记录。重启服务的时候就会从RDB和AOF两部分恢复数据，既保证数据完整又提高恢复数据性能

开启方式

配置项aof-use-rdb-preamble yes表示开启（默认）
