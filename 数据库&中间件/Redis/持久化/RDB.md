
以指定的时间间隔执行数据集的时间点快照，就是把某一时刻的数据和状态以文件的形式写在磁盘上。这样来即使故障宕机，快照文件不会丢失，数据的可靠性也得到保证（dump.rdb），恢复时将硬盘快照直接读回内存。

保存备份时执行全量快照，即把内存中的所有数据都记录在磁盘中

# 恢复

将备份文件（dump.rdb）移动到redis安装目录并启动服务即可

执行flushall/flushdb也会产生dump文件，但是是空的，没有意义

备注：不可以把备份文件dump.rdb和生产redis服务器放在同一台机器，必须分开各自存储，以防生产机物理损坏后备份文件也挂了。

# 触发RDB快照的场景

1. 配置文件中默认的快照配置（自动）
2. 手动执行save/bgsave命令
3. 执行flushall/flushdb产生空的dump.rdb文件
4. 执行shutdown且没有设置开启AOF持久化
5. 主从复制时，主节点自动触发

# 禁用快照

1. 动态停止RDB保存规则：`redis-cli config set save ""`
2. 去配置文件中修改 save ""

# 优化参数

* stop-writes-on-bgsave-error 默认yes，配置为no表示不在乎数据不一致或有其它手段发现和控制这种不一致，那么在快照写入失败时，确保redis继续接收新的写请求不中断
* rdbcompression 默认yes, 对磁盘中的快照是否进行压缩存储。LZF算法压缩。如果不想消耗CPU进行压缩，可以关闭
* rdbchecksum 默认yes，存储快照后，使用CRC64算法进行数据校验，会消耗性能。想要提升性能可以关闭
