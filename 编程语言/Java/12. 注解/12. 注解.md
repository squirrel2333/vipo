1. 注解，或者叫注释，单词为Annotation
2. 注解是一种引用数据类型，编译之后生成xxx.class文件
3. 怎么自定义注解？语法格式？

[修饰符列表] @interface 注解类型名{}

4. 注解怎么使用，用在什么地方
1. 注解使用时语法格式是：@注解类型名
2. 注解可以出现在类、属性、方法、变量上等，注解还可以出现在注解类型上

注解可以存在任意位置

```java
//注解修饰注解
@OtherAnnotation
public @interface MyAnnotation {}
```

```java
@MyAnnotation
public class AnnotationTest01 {
    @MyAnnotation
    private int no;

    @MyAnnotation
    public AnnotationTest01(){}

    @MyAnnotation
    public static void m1(){
        @MyAnnotation
        int i = 100;
    }

    @MyAnnotation
    public void m2(@MyAnnotation String name, @MyAnnotation int k){

    }
}

@MyAnnotation
interface MyInterface{}

@MyAnnotation
enum Season{
    SPRING,SUMMER,AUTUMN,WINTER
}
```
* Override只能注解方法，是给编译器参考的，和运行阶段没有关系
* 凡是java中带有Override注解，编译器都会进行编译检查，如果这个方法不是重写父类的方法，编译器报错

## Java.lang包下的注释类型

* Deprecated 用 @Deprecated注释的程序元素
* Override 表示一个方法声明打算重写超类中的另一个方法声明

```java
public class AnnotationTest03 {

    @Deprecated // 注解标注的元素已过时
    public void doSome(){
        System.out.println("do something");
    }
    public void doOther(){
        System.out.println("do other...");
    }
}
class T {
    public static void main(String[] args) {

    }

```
## 元注解

* 用来标注“注解类型”的注解
* 常见的元注解
+ Target
+ Retention
+ Documneted 表明这个注解应该被 javadoc工具记录。
+ Inherited 允许子类继承父类中的注解
* 关于Target注解：

这是一个元注解，用来标注“注解类型”的“注解”

这个Target注解用来标注“被标注的注解”可以出现在哪些位置上

`@Target(ElementType.METHOD)`：表示“被标注的注解”只能出现在方法上

`@Target(Value={CONSTRUCTOIR, FIELD, LOCAL\_VARIABLE, METHOD, PACKAGE, MODULE, PARAMETER, TYPE})`

* 关于Retention注解：

这是一个元注解，用来标注“注解类型”的“注解”

这个Retention注解用来标注“被标注的注解”最终保存在哪里

`@Rentention(RetentionPolicy.SOURCE)`//被标注的注解只保留在java源文件中

`@Rentention(RetentionPolicy.CLASS)`//被标注的注解只保留在class文件中

`@Rentention(RetentionPolicy.RUNTIME)`//被标注的注解只保留在class文件中，并且可以被反射机制所读取

```java
@Target(ElementType.METHOD)//被标注的注解只能出现在方法上
@Rentention(RetentionPolicy.SOURCE)//被标注的注解只保留在java源文件中
public @interface Override{
}
```
## 注解中定义属性

```java
public @interface MyAnnotation {
    /**
     * 我们通常在注解中可以定义属性，以下这个是MyAnnotation的name属性
     * 看着像一个方法，但实际上我们称之为属性name
     * @return
     */
    String name();
    String color();
    int age() default 25;//属性指定默认值
}
```

```java
public class MyAnnotationTest {
    //如果一个注解中有属性，必须给属性赋值(除非该属性使用default指定了默认值)
    @MyAnnotation(name="zhangsan", color = "yellow")
    public void doSome(){
    }
}
```
* 如果属性名为value，则在使用时该属性名value（且仅有一个）可以省略
* 属性类型只能是byte short int long float double boolean char String Class 枚举类型 以及以上任意一种的数组形式

```java
//如果属性是数组，使用时用大括号
@OtherAnnotation(age = 25, email = {"zhangsan@123.com","lisi@321.com"})
public void doSome(){}

//如果数组只有一个元素，可以省略大括号
@OtherAnnotation(age = 25, email = "zhangsan@123.com")
public void doSome(){}
```
再回看Deprecated的元注解

* @Retention(RetentionPolicy.RUNTIME) 中，省略了属性名value，属性类型为RetentionPolicy枚举类型，RUNTIME为其中之一
* @Target(value={CONSTRUCTOR, FIELD, LOCAL\_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE})

Target中value类型为ElementType[ ]，而ElementType为枚举类型

```java
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(value={CONSTRUCTOR, FIELD, LOCAL_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE})
public @interface Deprecated {
}
```
## 反射注解

```java
@Target({ElementType.TYPE, ElementType.METHOD})
// 希望可以被反射(必须是RUNTIME才能被反射机制读到)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
	String value() default "Shanghai";
}

public class MyAnnotationTest {
    @MyAnnotation
	public void doSome(){}
}

public class ReflectAnnotation {
    public static void main(String[] args) throws Exception {
        Class c = Class.forName("com.hqw.MyAnnotationTest"); // 拿到这个类
        Class stringClass = Class.forName("java.lang.String");
        // 判断类上是否有这个注解
        System.out.println(c.isAnnotationPresent(MyAnnotation.class)); // true
        System.out.println(stringClass.isAnnotationPresent(MyAnnotation.class)); //false
        // 获取注解
        if (c.isAnnotationPresent(MyAnnotation.class)){
            MyAnnotation myAnnotation = (MyAnnotation)c.getAnnotation(MyAnnotation.class);
        	// 获取注解对象的属性
            System.out.println(myAnnotation.value());
        }
    }
}
```
例子

```java
@Target(ElemenType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
	String username();
    String password();
}

public class MyAnnotationTest {
    @MyAnnotation(username = "hqw", password = "123")
    public void doSome() {}

    public static void main(String[] args) {
        Class c = Class.forName("com.hqw.MyAnnotationTest");
        // 获取doSome方法
        Method doSomeMethod = c.getDeclareMethod("doSome");
        // 判断是否存在注解
        if (doSomeMethod.isAnnotationPresent(MyAnnotation.class)) {
            MyAnnotation myAnnotation = doSomeMethod.getAnnotation(MyAnnotation.class);
        	System.out.println(myAnnotation.username());
        }
    }
}
```
## 注解的作用

需求：有一个注解Id, 这个注解只能在类上面，有这个注解的必须有一个int类型的id属性，如果没有就报异常

```java
@Target(ElemenType.TYPE)
// 必须被反射机制读取
@Retention(RetentionPolicy.RUNTIME)
public @interface Id {

}

@Id
public class User {
    int id;
    String name;
}

public class Test{
    public static void main(String[] args) throw Exception {
        Class userClass = Class.forName("come.hqw.User");
        if(userClass.getAnnotationPresent(Id.class) {
            //标记
            boolean isOk = false;
            // 必须存在int属性
            Field[] fields = userClass.getDeclaredFields();
            for(Field field : fields) {
                if ("id".equals(field.getName()) && "int".equals(field.getType().getSimpleName())) {
                    // 表示是合法
                    isOk = true;
                    break;
                }
            }
            // 判断是否合法
            if (!isOk) {
                throw new NoIdPropertyException("被@Id标注的类没有int类型的id属性");
            }
        }
   }
}
```
注解在程序中相当于标记
