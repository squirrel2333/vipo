# 测试替身技术

在软件测试中，尤其是单元测试中，"测试替身"（Test Doubles）是指用来替代真实对象或组件的辅助对象。常见的测试替身类型包括 Stub、Spy、Mock、Fake 等。它们有各自不同的用法和目的。下面是每种类型的详细介绍及其区别：

### 1. **Stub（桩）**

**Stub** 是最简单的测试替身。它的主要目的是提供一些预定义的行为或返回值，并且不关心对象是否被正确调用或其调用次数。通常，Stub 用来避免调用真实的外部依赖，比如数据库或网络请求。

#### 特点：

* 用于替换复杂的依赖对象，以提供固定的返回值。
* 不会验证行为，只提供预定的输出。
* 常用于数据源和外部系统替代。

#### 示例：

```java
java

复制代码
public class DatabaseStub extends Database {
    @Override
    public String getUserNameById(int id) {
        return "TestUser";  // 预定义返回值
    }
}
```
### 2. **Mock（模拟对象）**

**Mock** 是一种比 Stub 更复杂的测试替身，它不仅提供预定义的返回值，还会记录与其交互的行为。在 Mock 对象中，可以验证方法是否被调用了，以及调用了多少次，甚至可以验证调用时传递的参数是否正确。

#### 特点：

* 关注与对象交互的行为，而不是结果。
* 支持行为验证，可以检查方法调用次数、参数等。
* 使用场景包括验证某些依赖对象是否被正确调用。

#### 示例（使用 Mockito）：

```java

import static org.mockito.Mockito.*;

public void testWithMock() {
    List mockedList = mock(List.class);

    mockedList.add("one");
    verify(mockedList).add("one");  // 验证方法被调用
}
```
### 3. **Spy（间谍对象）**

**Spy** 是对真实对象的部分替代，它既可以调用真实对象的行为，也可以对特定行为进行伪造。Spy 对象是部分 Mock，允许我们监视真实对象的行为，同时可以选择对某些方法进行替换。

#### 特点：

* 对真实对象进行包装，既可调用真实方法，也可伪造某些行为。
* 允许验证行为，同时还可以调用真实方法。
* 用于希望保留部分对象真实行为的场景。

#### 示例（使用 Mockito）：

```java
import static org.mockito.Mockito.*;

public void testWithSpy() {
    List list = new ArrayList();
    List spyList = spy(list);

    spyList.add("one");
    spyList.add("two");

    verify(spyList).add("one");  // 验证行为
    assertEquals(2, spyList.size());  // 真实方法被调用
}
```
### 4. **Fake（伪对象）**

**Fake** 是一个比 Stub 更复杂的测试替身，它有时提供一个功能完备的轻量实现，而不是仅仅提供预定义的返回值。Fake 通常用来替代诸如数据库之类的真实系统，用以减少测试依赖，但其行为更接近真实对象。

#### 特点：

* 提供轻量但功能齐全的替代实现。
* 常用于构建简单版本的外部系统，如内存数据库或文件系统。
* 用于更复杂的测试场景，提供完整功能而非仅仅返回固定值。

#### 示例：

```java
public class FakeDatabase extends Database {
    private Map<Integer, String> data = new HashMap<>();

    @Override
    public String getUserNameById(int id) {
        return data.getOrDefault(id, "");  // 提供一个简化的数据库实现
    }

    public void addUser(int id, String name) {
        data.put(id, name);
    }
}
```
### 5. **Dummy（哑对象）**

**Dummy** 是用来满足方法签名要求的对象，但不会被实际使用。通常，它不会被调用到，或者即使被调用，它的行为也无关紧要。

#### 特点：

* 只用来满足参数列表，方法内不会使用它。
* 不会有任何行为，只是一个空壳。

#### 示例：

```java
public void testMethod() {
    SomeService service = new SomeService();
    service.performAction(null);  // 这里的 null 就是 Dummy，满足签名要求
}
```
### 总结

|  |  |  |
| --- | --- | --- |
| 类型 | 作用与特点 | 是否验证行为 |
| **Stub** | 提供预定义返回值，不关心调用细节 | 否 |
| **Mock** | 记录并验证方法调用次数、参数等行为 | 是 |
| **Spy** | 部分替代真实对象，既可以监控真实行为，也可以伪造某些行为 | 是 |
| **Fake** | 提供一个轻量的功能齐全的实现替代复杂依赖，常用于测试内部系统 | 否 |
| **Dummy** | 仅用于满足参数需求，不会被真正使用或验证 | 否 |

这些测试替身帮助我们在测试中隔离依赖，提高测试的效率和可靠性。选择合适的测试替身可以大大简化测试逻辑和复杂性。
