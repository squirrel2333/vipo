
程序在运行过程中, 非常可能遇到异常问题, Exception

* 检查性异常: 用户错误或问题引起, 是程序员无法预见的. 例如打开不存在的文件, 这些异常在编译时不能被简单地忽略
* 运行时异常: 可被程序员避免的异常. 可以在编译时被忽略
* 错误ERROR: 脱离程序员控制的问题.例如栈溢出

java可以把异常当作对象来处理, 并定义一个基类Java.lang.Throwable作为所有异常的超类

Java API中定义了许多异常类, 分为两大类, 错误Error和异常Exception

## Error

Error类对象由java虚拟机生成并抛出

当JVM不再有继续执行操作所需的内存资源时,将出现OutOfMemoryError. 这些异常发生时, Java虚拟机一般会选择线程终止

* StackOverflowError
* OutOfMemoryError

## Exception

有一个重要的子类RuntimeException(运行时异常)

* ArrayIndexOutOfBoundsException(数组下标越界) 非检查性
* NullPointerException (空指针异常) 非检查性
* ArithmeticException(算数异常) 非检查性
* MissingResourceException(丢失资源)
* ClassNotFoundException(找不到类) 检查性

这些异常时不检查异常,通常由程序逻辑引起

Error和Exception的区别: Error通常是灾难性的致命错误, 是程序无法控制和处理的, 当出现时Java虚拟机一般会选择终止线程; Exception通常可以被程序处理, 并在程序中应该尽可能的去处理这些异常

* 运行时异常：不需要提前处理，发生概率低
* 编译时异常：需处理，发生概率高
* 运行时异常和编译时异常都发生在运行阶段！因为只有在运行时才会new异常对象

## 异常处理机制

* Java对异常有两者处理方式：
+ 使用throws关键字，抛给调用者处理
- 一般不建议在main方法上使用throws，因为异常真的发生，JVM会停止运行。只要异常没有捕捉，采用上报的方式，此方法的后续代码不会执行
- 抛异常的父对象也可以
- 可以抛多个异常用逗号隔开
+ try...catch进行捕捉
- try语句块中的某一行出现异常，该行后面的代码不会执行（语句块中)
* try...catch
+ catch后面小括号可以是具体的异常类型，也可以是该异常类型的父类型
+ catch可以写多个，建议catch的时候，精确到一个一个处理。利于调试
+ catch写多个的时候，从上到下，必须遵守小到大
* 如果希望使用者处理，使用throws
* 子类不能比父类抛出更多的异常！！！

### 处理异常案例

```java
public class Test {
    public static void main(String[] args) {

        int a = 1;
        int b = 0;

        /*
        	方式一：
            程序底层new了一个ArithmeticException异常对象，抛给了main方法，main没有处理抛给了JVM，JVM停止运行

        try{ //try监控区域
            System.out.println(a/b);

        }
```

### try-with-resources

ARM(Automatic Resource Management)

数据流会在 try 执行完毕后自动被关闭，前提是，这些可关闭的资源必须实现 java.lang.AutoCloseable 接口

```java
private static void customBufferStreamCopy(File source, File target) {
    try (InputStream fis = new FileInputStream(source);
        OutputStream fos = new FileOutputStream(target)){

        byte[] buf = new byte[8192];

        int i;
        while ((i = fis.read(buf)) != -1) {
            fos.write(buf, 0, i);
        }
    }
    catch (Exception e) {
        e.printStackTrace();
    }
}
```

```java
FileInputStream fis = null;
try {
    fis = new FileInputStream("");
    String s = null;
    s.toString();//空指针异常
    //fis.close(); 异常时这句不会执行了，流关不了
} catch (FileNotFoundException e) {
    e.printStackTrace();
} catch (IOException e){
    e.printStackTrace();
} catch (NullPointException e){
    e.printStackTrace();
} finally {
    //流的关闭放在这里
    if (fis != null){ //避免fis空指针异常
        fis.close();
    }
}
```
**注意：不要在 finally 语句块中使用 return!** 当 try 语句和 finally 语句中都有 return 语句时，try 语句块中的 return 语句会被忽略。这是因为 try 语句中的 return 返回值会先被暂存在一个本地变量中，当执行到 finally 语句中的 return 之后，这个本地变量的值就变为了 finally 语句中的 return 返回值。

```java
private static int get() {
    int a = 1;
    try {
        int b = 3 / 1;
    } catch (ArithmeticException e) {
        return a = 8;
    } finally {
        a++;
    }
    return a;
} // 2

private static int get() {
   int a = 1;
   try {
       int b = 3 / 1;
       return a = 99;
   } catch (ArithmeticException e) {
       return a = 8;
   } finally {
       a++;
   }
} // 99, 最后a++不会赋给a进行return

private static int get() {
    int a = 1;
    try {
        int b = 3 / 0;
        a++;
    } catch (ArithmeticException e) {
        return a = 8;
    } finally {
        a++;
    }
    return a;
} // 8

private static int get() {
    int a = 1;
    try {
        a++;
        return a;
    } catch (ArithmeticException e) {

    } finally {
        return a = 666;
    }
} // 666
```

```java
package com.bjpowernode.javase.exception;

public class MyStack {

    private Object[] elements;

    private int index;

    public MyStack() {
        this.elements = new Object[10];
        this.index = -1;
    }

    /**
     * 压栈的方法
     * @param obj 被压入的元素
     */
    public void push(Object obj) throws MyStackOperationException {
        if(index >= elements.length - 1){
            // 改良之前
            //System.out.println("压栈失败，栈已满！");
            //return;

            // 创建异常对象
            //MyStackOperationException e = new MyStackOperationException("压栈失败，栈已满！");
            // 手动将异常抛出去！
            //throw e; //这里捕捉没有意义，自己new一个异常，自己捉，没有意义。栈已满这个信息你需要传递出去。

            // 合并（手动抛出异常！）
            throw new MyStackOperationException("压栈失败，栈已满！");
        }
        index++;
        elements[index] = obj;
        System.out.println("压栈" + obj + "元素成功，栈帧指向" + index);
    }

    /**
     * 弹栈的方法，从数组中往外取元素。每取出一个元素，栈帧向下移动一位。
     * @return
     */
    public void pop() throws MyStackOperationException {
        if(index < 0){
            //System.out.println("弹栈失败，栈已空！");
            //return;
            throw new MyStackOperationException("弹栈失败，栈已空！");
        }
        // 程序能够执行到此处说明栈没有空。
        System.out.print("弹栈" + elements[index] + "元素成功，");
        // 栈帧向下移动一位。
        index--;
        System.out.println("栈帧指向" + index);
    }
}
```
