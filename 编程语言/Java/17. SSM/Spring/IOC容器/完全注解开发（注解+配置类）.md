## 完全注解开发

以上的方式，第三方的类和扫描，仍需要依赖配置文件

完全不使用spring配置文件，写一个配置类来代替配置文件

1. 创建配置类，代替配置xml文件

```java
@Configuration // 作为配置类，替代xml配置
@ComponentScan(basePackages = "com.hqw.spring") // 扫描
public class SpringConfig {
}
```
2. 编写测试类

```java
@Test
public void test4() {
    // !!!!
    ApplicationContext context = new AnnotationConfigApplicationContext(SpringConfig.class);
    UserService us = context.getBean("userService", UserService.class);
    System.out.println(us);
    us.add();
}
```
## @Bean

```java
@Configuration
@ComponentScan("com.hqw.spring")
@PropertySource("classpath:jdbc.properties") //读取配置文件
@Import(value={JavaConfigurationB.class})// 可以导入其它配置类,实例化就不用单独实例化B了
public class JavaConfiguration {

    @Value("${jdbc.url}")
    private String url;
    @Value("${jdbc.driverClassName}")
    private String className;
    @Value("${jdbc.username}")
    private String username;
    @Value("${jdbc.password}")
    private String password;

    @Bean("haha") //默认bean名为方法名
    public DruidDataSource dataSource() {
        DruidDataSource druidDataSource = new DruidDataSource();
        druidDataSource.setUrl(url);
        druidDataSource.setDriverClassName(className);
        druidDataSource.setUsername(username);
        druidDataSource.setPassword(password);
        return druidDataSource;
    }

    @Bean
    // 写在参数中，会根据类型和名称帮助自动注入DruidDataSource
    public JdbcTemplate jdbcTemplate(DruidDataSource haha) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.setDataSource(haha);
        return jdbcTemplate;
    }
}
```
