# 可迭代对象Iterable

1. 支持的类型: str、list、tuple、dict、set
2. 可迭代的条件
1. 对象实现了`\_\_iter\_\_()`方法
2. `\_\_iter\_\_()`方法**必须返回迭代器对象**

```python
from collections.abc import Itrerable

print(isinstance("123", Iterable))
```
只要是可迭代对象，无论是否有下标，都可以使用for循环来进行迭代

# 迭代器Iterator

可以记住遍历位置的对象

迭代器条件:

1. 实现了`\_\_next\_\_()`和`\_\_iter\_\_()`方法

```python
li = [1, 2, 3, 4, 5]
lit = iter(li)
print(next(lit))
print(next(lit))
print(next(lit))
print(next(lit))
# 取完之后,再使用next时会引发StopIteration异常
```
* 步骤:
1. `iter()`调用对象的`\_\_iter\_\_()`, 并把`\_\_iter\_\_()`方法的返回结果作为自己的返回值
2. `next()`调用对象的`\_\_next\_\_()`, 一个一个的取元素
* 可迭代对象不一定是迭代器对象, 迭代器对象一定是可迭代对象
* 可迭代对象可以通过`iter()`方法转换成迭代器对象
* for循环的原理：
1. 先通过`\_\_iter\_\_()`获取可迭代对象的迭代器
2. 对获取到的迭代器对象不断地调用`\_\_next\_\_()`方法来获取下一个值并将其赋值给临时变量i
* `dir()`查看对象中的属性和方法

## 自定义迭代器

```python
class MyIterator(object):

    def __init__(self):
        self.num = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.num < 10:
            self.num += 1
            return self.num
        else:
            raise StopIteration # 迭代结束的标志，for循环会自动处理该异常，并结束循环

my_iter = MyIterator()
for i in my_iter:
    print(i)
```
一边做循环一边计算的机制, 是一种简便产生迭代器的手段

# 生成器

## 列表推导式

```python
li = [i * 5 for i in range(5)]
lt = (i * 5 for i in range(5))
print(li) # [0, 5, 10, 15, 20]
print(lt) # <generator object <genexpr> at 0x000001D27D79A4D0>
print(next(lt)) # 0

```
## 生成器函数

`yield`

1. 类似return, 将一个值或多个值返回给调用者
2. 遇到`yield`语句一次就返回一个结果, 在每个结果的中间挂起函数, 执行`next()`, 再重新从挂起点继续往下执行

```python
def gen(n):

    for i in range(n):
        yield i

for i in gen(10):
    print(i)
```
只要函数中有`yield`那么这个函数对象就变成了generator类型

```python
def fib(max):
    n, a, b = 0, 0, 1
    while n < max:
        yield b
        a, b = b, a + b
        n = n + 1
    return 'done'
```
for循环中，拿不到生成器函数对象的返回值。如果想要拿到generator的return返回值，就必须捕获StopIteration错误，返回值包含在StopIteration的value中：

```python
>>> g = fib(6)
>>> while True:
...     try:
...         x = next(g)
...         print('g:', x)
...     except StopIteration as e:
...         print('Generator return value:', e.value)
...         break
```
# 总结

包含关系: 可迭代对象 > 迭代器 > 生成器

凡是可作用于`for`循环的对象都是`Iterable`类型；

凡是可作用于`next()`函数的对象都是`Iterator`类型，它们表示一个惰性计算的序列；

集合数据类型如`list`、`dict`、`str`等是`Iterable`但不是`Iterator`，不过可以通过`iter()`函数获得一个`Iterator`对象。

Python的`for`循环本质上就是通过不断调用`next()`函数实现的，例如：

```python
for x in [1, 2, 3, 4, 5]:
    pass
```
实际上完全等价于：

```python
# 首先获得Iterator对象:
it = iter([1, 2, 3, 4, 5])
# 循环:
while True:
    try:
        # 获得下一个值:
        x = next(it)
    except StopIteration:
        # 遇到StopIteration就退出循环
        break
```
再直白一点：

通常情况下（例如列表、字典）：

1. **Iterable（可迭代对象）**：有 `\_\_iter\_\_`，**没有**`\_\_next\_\_`。
2. **Iterator（迭代器）**：有 `\_\_iter\_\_`，**也有**`\_\_next\_\_`。

**关键点在于：** 可迭代对象的 `\_\_iter\_\_` 方法，负责**创建一个新的、独立的**迭代器对象并返回。

这就好比：

* **可迭代对象**是“**书**”（内容是固定的）。
* **迭代器**是“**书签**”（记录读到哪一页了）。
* 书本身不需要记录读到哪了（它没有 `\_\_next\_\_`），但是当你想要读书时，书会给你发一个书签（`return Iterator`）。

```python
# --- 角色 1: 迭代器 (干苦力的书签) ---
class MyIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0  # 只有迭代器才需要记录“当前位置”

    def __iter__(self):
        return self

    def __next__(self):
        # 如果越界了，就报错停止
        if self.index >= len(self.data):
            raise StopIteration

        # 返回当前元素，并将指针后移
        value = self.data[self.index]
        self.index += 1
        return value

# --- 角色 2: 可迭代对象 (老板/容器) ---
class MyList:
    def __init__(self, *args):
        self.items = list(args)

    # 1. 我有 __iter__
    def __iter__(self):
        print("MyList: 有人要遍历我，我得雇佣一个新的迭代器...")
        # 2. 我返回了一个【别的对象】，那个对象才是真正的迭代器
        return MyIterator(self.items)
```
3. 虽然上面的原理很重要，但在实际工作中，我们很少写 `MyIterator` 那个类。我们会直接在 `\_\_iter\_\_` 里用 `yield`，Python 会自动帮我们生成那个迭代器对象。

```python
class MySmartList:
    def __init__(self, *args):
        self.items = list(args)

    def __iter__(self):
        # 这里的代码逻辑，本质上就是一个隐藏的迭代器类
        for item in self.items:
            yield item

# 效果和上面写两个类是一模一样的，但代码少了很多
```
