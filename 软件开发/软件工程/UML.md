
# UML基本介绍

1. 统一建模语言，是一种用于软件系统分析和设计的语言工具
2. 本身是一套符号规定，用于描述软件模型中各个元素和他们之间的关系，比如类、接口、实现、泛化、依赖、组合、聚合等

## UML图

1. 用例图
2. 静态结构图：类图、对象图、包图、组件图、部署图
3. 动态行为图：交互图（时序图或协作图）、状态图、活动图

### UML类图

1. 用于描述系统中的类（对象）本身的组织和类（对象）之间的各种静态关系。
2. 类之间的关系：依赖、泛化（继承）、实现、关联、聚合、组合

#### 依赖关系（Dependence）

只要在类中用到了对方，他们就是依赖关系（成员变量 返回类型 局部变量 ....)

![](UML_001.svg)* PlantUML中用 ..>表示依赖。指向被依赖的类
* 带箭头的虚线

#### 泛化关系（generalization）

实际上就是继承关系，是依赖的特例，is-a的关系

![](UML_002.svg)* 用 --|>表示，箭头从子类指向父类
* 带空心三角形的直线

#### 实现关系（implementation）

实际就是A类实现B类，他是依赖关系的特例

![](UML_003.svg)* 用..|>表示，箭头从实现类指向接口
* 虚线和带空心的三角形

#### 关联关系（Association）

类与类之间的联系。表示一个类和另一类是一种包含关系。用带箭头的实线表示，箭头指向被包含类（箭头从使用类指向被关联的类）。分为：1..1 表示另一个类的一个对象只与该类的一个对象有关系，0..\*表示另一个类的一个对象与该类的零个或多个对象有关系，1..\* 表示另一个类的一个对象与该类的一个或多个对象有关系，0..1 表示另一个类的一个对象没有或只与该类的一个对象有关系，\* 任意多个对象关联。

![](UML_004.svg)* 用 --> 表示
* 用带箭头的实线表示，箭头指向被包含类（箭头从使用类指向被关联的类）

#### 聚合关系（Aggregation）

是关联关系的特例，表示整体和部分的关系，has-a的关系，**整体与部分可以分开**

![](UML_005.svg)* 用 --o 表示
* 带空心菱形的直线。菱形从局部指向整体

#### 组合关系（Composition）

也是整体与部分的关系，一种强烈的包含（拥有）关系，**但整体与部分不可以分开**，整体的生命周期结束也就意味着部分的生命周期结束。此时部分需在整体的构造方法中创建。contains-a（has-a）

![](UML_006.svg)* 用--\*表示
* 组合关系用带实心菱形的直线表示，菱形从局部指向整体

```
public class Person {
    private IDCard card; //聚合
	private Head head = new Head(); //组合关系
}
```
但是如果Person实体中定义了对IDCard进行级联删除，即删除Person就连同IDCard一起删除，那么就是组合关系了
