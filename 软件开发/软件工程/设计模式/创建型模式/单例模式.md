# 介绍

对某个类只能存在一个对象实例，且该类只提供一个取得其对象实例的方法（静态方法）

# 饿汉式（静态常量）

## 实例

1. 构造器私有化 （防止new）
2. 类的内部创建对象
3. 向外暴露一个静态的公共方法 （getInstance)
4. 代码实现

```
public class SingletonTest {
    public static void main(String[] args) {
        Singleton singleton = Singleton.getInstance();
        Singleton singleton1 = Singleton.getInstance();
        System.out.println(singleton == singleton1); //true
    }
}

class Singleton {
    private Singleton() {}
    private final static Singleton instance = new Singleton();
    public static Singleton getInstance() {
        return instance;
    }
}
```
优点：写法简单，类装载就完成实例化，避免线程同步问题

缺点：没有达到懒加载的效果，如果没用过这个实例，就造成了内存浪费

结论：可用，但可能造成内存浪费

# 懒汉式（线程不安全）

```
public class SingletonTest2 {
    public static void main(String[] args) {
        Singleton2 singleton = Singleton2.getInstance();
        Singleton2 singleton1 = Singleton2.getInstance();
        System.out.println(singleton == singleton1); //true
    }
}

class Singleton2 {
    private static Singleton2 instance;
    private Singleton2() {}
    //静态方法，当使用时，才去创建instance
    public static Singleton2 getInstance() {
        if (instance == null) {
            instance = new Singleton2();
        }
        return instance;
    }
}
```
优点：起到了懒加载的效果，但只在单线程下使用

缺点：如果在多线程下，一个线程进入了if (instance == null)还未继续执行，另一个也通过了此判断，就会产生多个实例。所以多线程不可用

结论：实际中不采用

# 懒汉式（同步方法）

```
public class SingletonTest2 {
    public static void main(String[] args) {
        Singleton2 singleton = Singleton2.getInstance();
        Singleton2 singleton1 = Singleton2.getInstance();
        System.out.println(singleton == singleton1); //true
    }
}

class Singleton2 {
    private static Singleton2 instance;
    private Singleton2() {}
    //静态方法，当使用时，才去创建instance
    //加入同步处理
    public static synchronized Singleton2 getInstance() {
        if (instance == null) {
            instance = new Singleton2();
        }
        return instance;
    }
}
```
优点：解决了线程不安全行为

缺点：效率低，每个线程要多的类实例时，都要进行一次同步，实际上只需要一次，后面的想要获得之间return

实际中不推荐

# 双重检查

```
public class SingletonTest2 {
    public static void main(String[] args) {
        Singleton2 singleton = Singleton2.getInstance();
        Singleton2 singleton1 = Singleton2.getInstance();
        System.out.println(singleton == singleton1); //true
    }
}

class Singleton2 {
    private static volatile Singleton2 instance;
    private Singleton2() {}
    //静态方法，当使用时，才去创建instance
    //双重检查，解决线程安全问题
    public static Singleton2 getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if(instance == null) {
                    instance = new Singleton2();
                }
            }
        }
        return instance;
    }
}
```
1. 保证了线程安全
2. 保证了延迟加载
3. 推荐使用

# 登记式/静态内部类

```
public class SingletonTest2 {
    public static void main(String[] args) {
        Singleton2 singleton = Singleton2.getInstance();
        Singleton2 singleton1 = Singleton2.getInstance();
        System.out.println(singleton == singleton1); //true
    }
}

class Singleton2 {
    private Singleton2() {}

    private static class SingletonInstance {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton2 getInstance() {
        return SingletonInstance.INSTANCE;
    }
}
```
1. 类装载时不会立即实例化，保证懒加载
2. 在调用getInstance时，装载静态内部类且只会加载一次，同时是线程安全的
3. 推荐使用

# 枚举方式

```
public class SingletonTest2 {
    public static void main(String[] args) {
        Singleton singleton = Singleton.INSTANCE;
        Singleton singleton1 = Singleton2.INSTANCE;
        System.out.println(singleton == singleton1); //true
        singleton.sayOK();
        singleton1.sayOK();
    }
}

enum Singleton {
    INSTANCE;
    public void sayOK() {
        System.out.println("OK");
    }
}
```
1. 借助枚举实现，避免线程问题，而且防止反序列化重新创建对象
2. 推荐

# 单例模式应用

JDK中，java.lang.Runtime就是饿汉式的单例模式

# 注意事项

1. 单例模式保证内存中只存在一个对象，节省了系统资源。对于需要频繁创建销毁的对象，使用单例模式提高性能
2. 当想实例化单例对象，必须使用相应的方法而不是new
3. 场景：频繁创建销毁、创建耗时过多或资源消耗较多但又经常使用的对象、工具类对象、频繁访问数据库或文件的对象（比如数据源、session工厂等）
